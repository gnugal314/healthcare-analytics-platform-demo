# Architecture Overview

## Purpose
This repository demonstrates a production-style healthcare analytics platform built in Python.

## Domains
- Claims analytics
- Pharmacy 340B analytics

## Pipeline layers
1. Raw ingestion
2. Staging
3. Transformation
4. Curated dimensional models
5. Analytics outputs
6. Data quality validation
7. Observability

## Design principles
- modular code organization
- clear separation of concerns
- trust in data through validation
- analytics-ready outputs
- healthcare-relevant business logic
