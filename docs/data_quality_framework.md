# Data Quality Framework

## Objective
Ensure data is complete, consistent, valid, and trustworthy before it reaches analytics outputs.

## Validation categories
- row count and completeness checks
- null checks on required keys
- duplicate key checks
- non-negative financial and quantity fields
- business rule validation for 340B savings eligibility

## Why this matters
In healthcare analytics, trust in the data is critical for reporting, operational decisions, and downstream analytics.
